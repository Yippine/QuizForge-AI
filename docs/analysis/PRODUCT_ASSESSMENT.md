## 二、服務的產業應用性

### 正面條件：

- IPAS 每年報名人數約 1~2 萬人次（iCAP + 數位部推動），是台灣資安領域的主流認證之一
- 目前市場上缺乏高品質的免費線上練習平台（主流仍是 PDF 或補習班講義）
- 政府持續推動 AI 法規、資安管理，考試需求只增不減
- 本應用的題庫品質（官方題 + 模擬題 + 解析）在市場上具有稀缺性

### 制約條件：

- 目前只覆蓋 AI 規劃師中級，完整 IPAS 有多個類組（資安、雲端、AIoT...），覆蓋率低
- 靜態架構限制了動態功能（即時出題、學員管理、題庫共享）
- 沒有用戶系統 = 沒有留存率數據 = 無法驗證真實用量

### 橫向延伸可能性（如果架構改造）：

- 其他證照：PMP、AWS、Google、CCNA，機制完全可複用
- B2B 賣給企業培訓部門（資安合規訓練）
- B2B 賣給補習班、職訓中心（教師端管理）

## 三、推廣難易度

### 容易的部分：

- 目標群體明確（IPAS 考生），可精準投放（PTT 資安版、社群、Dcard）
- 免費提供即是最好的獲客工具
- 現有品質已可口碑推薦

### 困難的部分：

- 沒有會員系統 = 無法積累忠實用戶。考生來一次刷題，下次換了裝置、或清了瀏覽器，進度歸零，黏著度極低
- 無分享機制：考生無法把自訂題組或學習進度分享給同學
- 推廣後無法回答「有多少人在用？考過的人占幾成？」，難以形成口碑資料

## 四、客群是誰，他們真正想要什麼

### 主要客群：在職人員 / 應屆考生（備考 IPAS）

#### 他們的核心行為模式：

碎片時間刷題 → 模擬考摸底 → 針對弱點複習 → 再考 → 過關

#### 他們最想要（按重要性排序）：

1. 快速進入刷題，不要多餘的點擊（1~2 步就到題目）
2. 進度能跨裝置保存（手機/電腦切換）
3. 智能弱點分析（哪個主題最弱、推薦複習）
4. 模擬考體驗逼真（計時、計分、結果報告）
5. 解析清楚、說明到位

#### 他們不在乎（或不主動需要）：

- KaTeX 數學公式渲染（IPAS 考試很少有數學式）
- Mermaid 架構圖（有用但不是核心動機）
- 5 層深的階層導航（過於複雜）
- 術語表系統（備考時通常直接背題目）
- TOC 側邊欄（刷題情境用不到）

### 次要客群：老師 / 補習班 / 企業培訓負責人

他們想要：
- 匯入自己的題庫
- 追蹤學員進度
- 分配題組給特定學員
- 一鍵產生測驗連結

## 五、設計偏移的根本問題

### 這是最關鍵的洞察。

**問題**：你在打造一個「學習資源中心」，但使用者需要的是「考試刷題工具」

應用名叫 QuizForge-AI，核心訴求應該是「刷題」，但大量開發精力花在了：

- 5層導航系統（L1~L5）
- Markdown 渲染 + Mermaid + KaTeX
- TOC 側邊欄（IntersectionObserver）
- 術語表（搜尋、分類、相關詞）
- 講義/手冊/筆記資源系統

這些功能本身沒有錯，但它們屬於「教材平台」的邏輯，而不是「刷題工具」的邏輯。

一個 IPAS 考生打開 app，心裡想的是：

> 「我要練 L23 機器學習主題的題目，看看今天能到幾分」

但目前的路徑是：

> 首頁 → 選科目 Modal → IPAS Overview → ai-planning → intermediate → L23 → Practice Hub → Topic Selection → 進入刷題

**6~7 步，才到達最核心的功能。**

## 七、會員登入：重不重要？

**結論：非常重要，是目前最大的產品缺口。**

### 沒有會員系統的代價：

1. 答題進度存 localStorage → 換裝置即清零，考生一旦換手機，要從頭開始
2. 無法做留存率分析 → 不知道有多少人回訪、用了幾次
3. 無法推廣社群功能 → 無法讓考生互相看對方的進度、分享
4. 無法收費 → 如果未來想做付費方案，沒有帳戶系統則無法落地

### 實作建議（最小化成本）：

- 用 Cloudflare Workers + D1（SQLite） 搭配 Google OAuth，不需要自建後端
- 或更輕量：用 Supabase，直接提供 Google/GitHub 登入 + 資料庫

---

## 八、一鍵匯入題庫：重不重要？

**結論：對特定客群非常重要，但不是第一優先。**

### 什麼情況下重要：

- 老師想把自己出的題目放進來
- 補習班想客製化一套練習系統
- 考生想把網路上找到的題目整合進來

### 為什麼不是第一優先：

- 核心用戶（考生）對「匯入」的需求遠低於「刷題品質」和「進度保存」
- 匯入功能需要後端支援，且格式驗證複雜，開發成本高
- 在有會員系統之前，匯入的題目也無法歸屬到個人帳號

### 建議順序：

會員系統 → 跨裝置進度同步 → AI 弱點推薦 → 一鍵匯入題庫

## 九、重新思考的方向建議

### 核心定位重整

**現況（模糊）：**
> 「IPAS 學習資源中心 + 刷題工具」

**建議（聚焦）：**
> 「最好用的 IPAS 考試練習 App，附帶學習資源」
>
> - 主線 = 刷題
> - 支線 = 學習資源（降為輔助功能）

### 優先順序建議

#### P0（立即要做）：

1. 簡化首頁入口 → 進入刷題不超過 2 步
2. 加上 Google 登入（Supabase or Cloudflare）
3. 進度雲端同步（localStorage 升級為後端）

#### P1（3個月內）：

4. 個人化弱點分析（哪個主題正確率最低，自動推薦複習）
5. 擴充題庫（L22 資安科目）
6. 簡化導航（5層壓縮為3層：科目 → 主題 → 刷題）

#### P2（6個月內）：

7. 一鍵匯入題庫（教師版）
8. 分享學習進度（社群功能輕量版）
9. PWA 離線支援

#### 可以暫緩或降低投入：

- KaTeX 公式（IPAS 用不太到）
- Mermaid 圖表新增（維護現有的即可）
- 術語表擴充（保持現有規模）
- 階層導航繼續深化（現有就夠了）

## 總結

QuizForge-AI 的技術品質和代碼架構相當紮實，354 題、Markdown 渲染、雙模式答題都是真實的競爭力。但目前的產品設計偏向「教材平台」，而真實用戶需要的是**「考試刷題工具」**。

### 最急迫的三件事：

1. **加會員登入**（進度跨裝置，是用戶留存的基礎）
2. **簡化入口**（首頁 → 刷題不超過 2 步）
3. **擴充題庫**（現在只覆蓋 AI 規劃師中級，覆蓋率太低）

一鍵匯入題庫有價值，但要排在會員系統之後。架構本身不需要大改，只需要在優先順序和入口動線上做重新梳理。
