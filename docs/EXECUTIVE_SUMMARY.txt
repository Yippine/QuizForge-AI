【Formula 結構分析】

每個 Formula 文件包含 6 大章節 (標準化結構)：

1. 核心定義 (20%) - 主題定義 + 核心概念 + CFDS 分解 + 技術定位
2. 關鍵公式 (25%) - 數學公式 + 技術演進 + 架構設計
3. 對比矩陣 (15%) - 技術對標 + 決策樹 + 選擇指南
4. 實務應用 (20%) - 應用場景 + 實作步驟 + 常見陷阱
5. 記憶口訣 (10%) - 考試技巧 + 易混淆辨析
6. 自我驗證 (10%) - 選擇題 + 簡答題 + 易錯點

📌 範例：L21101-自然語言處理技術與應用
   ├─ Transformer 公式：Attention(Q,K,V) = softmax(QK^T/√d_k)V
   ├─ 技術對比表：One-hot vs Word2Vec vs BERT (動靜態對比)
   ├─ 應用案例：智慧客服、情感分析、機器翻譯
   └─ 記憶法則：「理(NLU) 處(NLP) 生(NLG)」三步走

【題庫數據結構】

JSON 題目包含字段 (11 個)：

{
  "question_id": "L21101_001",        # 唯一標識
  "topic": "L21101",                   # Formula 編碼
  "difficulty": "simple/medium/hard",  # 難度級別
  "question_type": "technical_definition",  # 題型
  "question": "自然語言處理的核心目標為何...",  # 題敘
  "options": {
    "A": "選項A",
    "B": "選項B",
    "C": "選項C",
    "D": "選項D"
  },
  "answer": "B",                        # 正確答案
  "explanation": "詳細解析說明...",    # 解析
  "keywords": ["NLP", "定義", "語言處理"],  # 關鍵字
  "reference": "L21101-文件名 - 章節名"  # 參考來源
}

難度分布：
├─ simple (簡單)：約 30%
├─ medium (中等)：約 50%
└─ hard (困難)：約 20%

題型分布：
├─ technical_definition (技術定義)：約 25%
├─ technical_principles (技術原理)：約 30%
├─ technical_selection (技術選擇)：約 25%
└─ practical_application (實務應用)：約 20%

【編碼系統說明】

1. Formula 編碼：L21101
   L = Course (課程標記)
   2 = Level (級別：初級/中級)
   1 = Subject (科目：1=人工智慧, 3=機器學習)
   1 = Chapter (章節：1-4)
   01 = Topic (主題序號：01-99)

2. 題目編碼 (官方)：OFF_L21_CH3_001
   OFF = Official (來源：官方)
   L21 = Subject (科目1)
   CH3 = Chapter (第3章)
   001 = Sequence (序號)

3. 題目編碼 (模擬考)：L21101_001
   L21101 = Formula 編碼 (直接映射)
   001 = Sequence (序號)

【建議技術棧】

推薦選項 (基於項目規模與 Ollama 集成需求)：

前端：React / Vue 3
      ├─ UI：Tailwind CSS

      └─ 構建：Vite

後端：Python + FastAPI (推薦)
      ├─ 原因：便於 AI 集成

      ├─ 認證：JWT
      └─ ORM：SQLAlchemy

       ├─ 錯誤分析
       ├─ 學習推薦
       └─ 知識提取

基礎設施：Docker + GitHub Actions + 云部署
